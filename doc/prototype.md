prototype.md

Goal

Reincarnate the old OCaml IDL compiler as a small contract-first code generator targeting TypeScript/Bun.

The prototype must prove one thing:

A strict IDL can generate enough boring code that bootstrapping simple services is faster and cleaner than handwriting everything.

This is not a protobuf competitor, not a Rails generator, and not a workflow engine.

It is:
	•	a small IDL
	•	an OCaml parser + semantic compiler
	•	a TypeScript/Bun code generator
	•	autogenerated dumb scenarios
	•	handwritten custom logic where the domain matters

Core design principles
	1.	Strict contracts over shapeless flexibility
	2.	Generated boring parts, handwritten meaningful parts
	3.	Stable structure, faster-changing mounted logic
	4.	Small grammar
	5.	C-style blocks only
	6.	No indentation semantics
	7.	No manual field indexes in source
	8.	References are explicit via *Type
	9.	Annotations carry meaning, not hacks
	10.	Versioning/migration comes later, after MVP

Semantic model

struct

A struct is a shape.

Use it for:
	•	request bodies
	•	response bodies
	•	embedded value objects
	•	simple config shapes

Example:

struct Profile {
  string name;
  string headline;
}


⸻

enum

A closed symbolic set.

enum EmploymentType {
  FULL_TIME;
  CONTRACT;
  TEMPORARY;
}


⸻

*Type

*Type means a typed reference / handle / identity to an existing object of type Type.

It is not a raw pointer and has no memory semantics.

It may be represented as:
	•	UUID
	•	string reference
	•	opaque ID

Examples

*Profile
*Company
*Job

Rule:

*Type always means “reference to an existing object of type Type”.

⸻

service

A service describes the operation surface.

Operations may be explicitly declared or expanded from crud.

Custom business logic is handwritten behind generated stubs.

Example:

service Company {
  add(*Company company, Job job) *Job;
}

Grammar sketch

file := declaration*

declaration :=
    struct_decl
  | enum_decl
  | service_decl

struct_decl :=
    annotation* "struct" IDENT "{" field* "}"

field :=
    annotation* type IDENT ";"

enum_decl :=
    annotation* "enum" IDENT "{" enum_member* "}"

enum_member :=
    annotation* IDENT ";"

service_decl :=
    annotation* "service" IDENT "{" service_item* "}"

service_item :=
    annotation* method_decl
  | annotation* "crud" ";"

method_decl :=
    IDENT "(" param_list? ")" return_type ";"

param_list :=
    param ("," param)*

param :=
    type IDENT

return_type :=
    type

type :=
      "*" IDENT
    | IDENT
    | IDENT "<" type ("," type)* ">"
    | primitive

primitive :=
      string
    | int
    | float
    | bool
    | binary
    | void

Compiler pipeline

Parser

Input: IDL text

Output: AST

Responsibilities:
	•	tokenize
	•	parse syntax
	•	attach annotations
	•	preserve source locations

⸻

AST

AST mirrors syntax.

Example node types:
	•	struct_decl
	•	enum_decl
	•	service_decl
	•	method_decl
	•	field
	•	annotation
	•	type

Semantic compilation

Responsibilities:
	•	resolve named types
	•	validate *Type
	•	validate Map<string, primitive>
	•	expand crud
	•	detect duplicates
	•	attach annotation metadata

⸻

Handler interface generation

IDL:

service Company {
  crud;

  add(*Company company, Job job) *Job;
}

TypeScript:

export interface CompanyHandlers {

  create(data: Company): Promise<CompanyId>

  get(company: CompanyId): Promise<Company>

  update(company: CompanyId, data: Company): Promise<void>

  delete(company: CompanyId): Promise<void>

  list(): Promise<CompanyId[]>

  add(company: CompanyId, job: Job): Promise<JobId>

}
